# Font Awesome エラーの解決方法

## 問題

Font Awesome の CDN からアイコンを読み込む際にエラーが発生しています：
```
Error: <path> attribute d: Expected number
```

これは Google Apps Script の環境で外部 CDN リソースの読み込みに制限があるためです。

## 解決策

Font Awesome を使わないバージョンを作成しました。

### ステップ1: シンプル版のファイルをアップロード

GASエディタで **index_GAS_simple.html** を使用します：

1. GASエディタで、既存の `index_GAS` ファイルを削除（または名前変更）
2. 「+」→「HTML」で新しいファイルを作成
3. 名前を `index_GAS` にする
4. **index_GAS_simple.html** の内容をコピー＆ペースト

### ステップ2: Code.gs はそのまま

Code.gs は変更不要です。すでに `index_GAS` を参照しているので、上記で置き換えれば動作します。

### ステップ3: デプロイ

1. 保存
2. 「デプロイ」→「デプロイを管理」→ 編集アイコン
3. 「バージョン」→「新バージョン」
4. 「デプロイ」

### 変更点

Font Awesome のアイコンを以下のように置き換えました：

| 元のアイコン | 新しい表示 |
|------------|----------|
| `fa-calendar-heart` | 📅 (絵文字) |
| `fa-plus` | ➕ (絵文字) |
| `fa-chevron-left` | ◀ (Unicode) |
| `fa-chevron-right` | ▶ (Unicode) |
| `fa-xmark` | ✕ (Unicode) |
| `fa-spinner` | ローディングアニメーション (CSS) |

## 確認

デプロイ後、URLにアクセスして以下を確認：

1. ✅ ページが表示される
2. ✅ ローディング画面が表示されてから、カレンダーが表示される
3. ✅ ブラウザのコンソール（F12）にエラーが出ていない

## トラブルシューティング

### それでもエラーが出る場合

1. **ブラウザのキャッシュをクリア**
   - `Ctrl + Shift + Delete` (Windows)
   - `Cmd + Shift + Delete` (Mac)

2. **シークレットモードで開く**
   - Chrome: `Ctrl + Shift + N`
   - Edge: `Ctrl + Shift + P`

3. **Google Fonts もエラーが出る場合**
   - styles.css の先頭で Google Fonts のインポートをコメントアウト
   - フォントファミリーをシステムフォントに変更

## 次のステップ

エラーが解決したら、カレンダーの機能をテストしてください：

1. 「予定を追加」ボタンをクリック
2. タイトル、日付、カテゴリーを入力
3. 「保存」をクリック
4. カレンダーに予定が表示されることを確認
5. ページをリロードしても予定が残っていることを確認（スプレッドシートに保存されている）
